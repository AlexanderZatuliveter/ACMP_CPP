Привет! Давай разберем этот «умный массив». Если ты когда-нибудь мучился с обычными массивами в C++ (где нужно заранее знать размер и молиться, чтобы не выйти за его пределы), то `std::vector` станет твоим лучшим другом.

---

## Что это вообще такое?

`std::vector` — это часть **STL** (Standard Template Library, стандартная библиотека шаблонов). По сути, это **динамический массив**. 

В отличие от стандартного массива (`int arr[10]`), вектор умеет сам менять свой размер. Если в нем закончилось место, он просто «расширяется», переезжая в более просторную область памяти. Это избавляет тебя от необходимости вручную управлять выделением и освобождением памяти через `new` и `delete`.



### Что он дает?
1.  **Динамический размер:** Ты можешь добавлять элементы, пока не закончится оперативная память.
2.  **Безопасность:** Есть методы, которые проверяют, не вылез ли ты за границы массива.
3.  **Совместимость:** Элементы вектора лежат в памяти подряд (как в обычном массиве), поэтому он очень быстрый и дружит с «железом» (кэш-памятью).
4.  **Умное управление:** Он сам знает свой размер и умеет очищать за собой память при выходе из области видимости.

---

## Базовые операции

Для начала работы нужно подключить библиотеку: `#include <vector>`.

### 1. Создание и инициализация
```cpp
#include <vector>
#include <iostream>

int main() {
    std::vector<int> v1;               // Пустой вектор для целых чисел
    std::vector<int> v2(5, 10);        // Вектор из 5 элементов, каждый равен 10
    std::vector<int> v3 = {1, 2, 3};   // Инициализация списком (C++11 и выше)
    return 0;
}
```

### 2. Добавление и удаление элементов
Самая мощная фишка — добавление в конец. Это происходит очень быстро (в среднем за $O(1)$).

```cpp
v1.push_back(42); // Добавляет 42 в конец вектора
v1.pop_back();    // Удаляет последний элемент
v1.clear();       // Полностью очищает вектор (размер становится 0)
```

### 3. Доступ к элементам
Тут есть два основных пути:

* **`v[i]`** — работает быстро, как в обычном массиве, но **не проверяет границы**. Если обратишься к индексу 100 при размере 10 — программа может просто «упасть» или выдать мусор.
* **`v.at(i)`** — чуть медленнее, но если индекс неверный, он выбросит исключение. Это безопаснее при отладке.

### 4. Размер и емкость
Важно различать эти понятия:
* **`size()`** — сколько элементов сейчас реально лежит в векторе.
* **`capacity()`** — на сколько элементов зарезервировано место в памяти (чтобы не перевыделять её при каждом `push_back`).

---

## Шпаргалка по основным методам

| Метод | Описание |
| :--- | :--- |
| `push_back(x)` | Добавить элемент `x` в конец. |
| `size()` | Узнать текущее количество элементов. |
| `empty()` | Возвращает `true`, если вектор пуст. |
| `front()` / `back()` | Получить первый / последний элемент. |
| `resize(n)` | Изменить размер вектора до `n` элементов. |
| `reserve(n)` | Заранее забронировать место под `n` элементов (помогает ускорить работу). |

---

## Маленький совет «от профи»
Если ты заранее знаешь, что в векторе будет, скажем, 1000 элементов, всегда используй метод `v.reserve(1000)`. Это предотвратит лишние переезды вектора в памяти и сделает твой код ощутимо быстрее.
